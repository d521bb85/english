# Assessment Protocol

Use this protocol for the initial assessment and for major reassessments later.

## Inputs

Required files:

- `learner/submissions/00-Initial-Interview-Answers.md`
- `learner/submissions/00-Baseline-Writing-Diagnostic-Response.md`

Reference files:

- `learner/intake/Initial-Interview.md`
- `learner/intake/Baseline-Writing-Diagnostic.md`
- `facilitator/Evaluation-Rubric.md`

## Assessment Steps

1. Read the learner interview.
2. Read the diagnostic writing carefully as raw evidence.
3. Estimate the learner's current writing band on the CEFR scale.
4. Identify strengths, recurring weaknesses, and the most urgent bottlenecks.
5. Separate first-draft performance from self-editing ability when possible.
6. Create a baseline feedback file in `learner/feedback/`.
7. Update the long-term tracking files in `progress/`.
8. Create the first post-assessment assignment in `learner/assignments/`.
9. Update `learner/current/Current-Focus.md` to reflect the new phase.

## Required Assessment Output

The baseline feedback should include:

- estimated CEFR writing level
- confidence level and why
- strengths
- main error patterns
- recommended focus for the next 2 to 4 weeks
- the single most important habit for the learner to adopt

## Roadmap Rules

After assessment:

- set a working current band
- define the next milestone band
- choose 2 to 4 focus areas only
- avoid promising a date for C2 without evidence and study consistency

## Reassessment Triggers

Run a structured reassessment when:

- the learner completes a major phase
- performance becomes clearly stronger than the current roadmap
- the learner plateaus for several weeks
- the learner requests a formal level check

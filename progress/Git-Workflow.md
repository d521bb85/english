# Git Workflow

This repository doubles as a study log. Use git to preserve evidence of progress.

## Suggested Commit Types

- `intake:` for interview and diagnostic work
- `submission:` for new writing tasks
- `feedback:` for teacher evaluations
- `revision:` for revised drafts
- `roadmap:` for major progress updates
- `review:` for weekly or phase reviews

## Example Commit Messages

- `intake: complete initial interview`
- `intake: submit baseline writing diagnostic`
- `feedback: baseline assessment and first roadmap`
- `submission: write opinion paragraph on AI tools`
- `revision: improve paragraph cohesion in task 02`

## Practical Rule

Make one commit per meaningful learning step. That gives you a clean history of how your writing changed over time.

## Minimal Command Flow

```bash
git status
git add learner/ progress/
git commit -m "submission: complete task 01 draft"
```

## What To Commit

Commit:

- assignments
- submissions
- feedback
- progress updates

Avoid relying only on chat history. The repository should remain the durable record.
